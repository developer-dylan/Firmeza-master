# Firmeza.Client

**Overview**

Firmeza.Client is the frontend Single Page Application (SPA) for the Firmeza project. It is built with Vue 3, Vite and Tailwind CSS and provides the public/user-facing interface for catalog browsing, cart management, authentication and checkout (consuming the `Firmeza.Api` or `Firmeza.Web` endpoints).

**Project structure**

- `index.html` - SPA entry point.
- `src/` - source files:
  - `main.js` - app bootstrap and router/pinia setup.
  - `App.vue` - root Vue component.
  - `components/` - reusable Vue components (cart drawer, modals, etc.).
  - `views/` - page-level Vue components (Catalog, Cart, Login, Register).
  - `router/` - Vue Router configuration.
  - `services/api.js` - Axios instance and API helpers.
  - `stores/` - Pinia stores (auth, cart).
  - `style.css` - Tailwind entry styles.
- `public/` - static assets.
- `wwwroot/` and `Views/` - integration with the ASP.NET app when used server-side.

**Technologies and tools**

- **Framework**: Vue 3 (Composition API).
- **Build tool**: Vite.
- **Styling**: Tailwind CSS + PostCSS.
- **State**: Pinia.
- **HTTP client**: Axios.
- **Dev tools**: Node.js (recommended >= 18), npm or yarn.

**Prerequisites**

- Node.js (recommended LTS >= 18) and npm.
- If consuming local API: `Firmeza.Api` or `Firmeza.Web` must be running and reachable.

**Configuration**

- API base URL is set in `src/services/api.js`. You can configure it via environment variables when building with Vite. Example environment files:

  - Create `.env` (or `.env.development`) with:
    - `VITE_API_BASE_URL=http://localhost:7000` (adjust port/host accordingly)

- Vite exposes `import.meta.env.VITE_API_BASE_URL` to the client code.

**Available scripts**

From the `Firmeza.Client` folder run:

```bash
npm install
npm run dev      # start development server (Vite)
npm run build    # build production assets
npm run preview  # preview production build
```

Scripts defined in `package.json`:

- `dev`: `vite`
- `build`: `vite build`
- `preview`: `vite preview`

**Development workflow**

1. Install dependencies:

```bash
npm install
```

2. Start the dev server:

```bash
npm run dev
```

3. Open the URL shown by Vite (usually `http://localhost:5173`) and use the app. Ensure the API is reachable and `VITE_API_BASE_URL` points to it.

**Production build and integration**

- Build the static assets:

```bash
npm run build
```

- The `dist/` folder generated by Vite can be served by any static server or integrated into the ASP.NET app (`wwwroot/`) if needed.

**Notes**

- The frontend expects an API that serves JSON and uses JWT or cookie-based authentication depending on how you wire `Firmeza.Api` / `Firmeza.Web`.
- Update CORS and API base URL when running across different hosts.

**Useful files**

- `src/services/api.js` - Axios setup and base URL configuration.
- `src/stores/auth.js` - authentication store.

**Deployment tips**

- Use environment variables to set `VITE_API_BASE_URL` during CI/CD.
- Serve built assets from a CDN or from the ASP.NET `wwwroot` folder for integrated deployments.
